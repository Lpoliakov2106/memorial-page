## Семейное древо (в разработке)

### Описание
Визуализация родственных связей в виде интерактивного дерева. Позволяет отображать связи между людьми, чьи страницы памяти созданы в системе.

### Структура данных
Структура данных уже поддерживается в localStorage:

```javascript
relatives: [
  {
    id: "rel_1",
    name: "Пётр Николаевич Смирнов",
    relation: "партнёр",  // тип отношения
    years: "1943–2020",
    photo: null,
    note: "Супруг"
  },
  {
    id: "rel_2",
    name: "Елена Петровна Волкова",
    relation: "дочь",
    years: "1971",
    photo: null,
    note: ""
  }
]
```

### Типы отношений
- `родитель` — отец/мать
- `ребёнок` — сын/дочь
- `партнёр` — супруг(а)
- `брат/сестра` — братья и сёстры
- `дед/бабушка` — дедушка/бабушка
- `внук/внучка` — внуки
- `другое` — другие родственники

### Планируемый функционал
1. **Отдельная вкладка "Семейное древо"** на странице профиля
2. **Визуализация с D3.js или Canvas** — интерактивное дерево с возможностью зума и перемещения
3. **Кликабельные узлы** — переход к странице памяти родственника, если она существует
4. **Редактор связей** — добавление/удаление родственников через модальное окно
5. **Связывание страниц** — возможность связать существующую страницу памяти как родственника

### Пример интерфейса (планируется)
```
         [Бабушка]  [Дедушка]
              \      /
               [Мать]——[Отец]
                   |
        ┌─────────┼─────────┐
        |         |         |
     [Брат]   [ТЕКУЩИЙ]  [Сестра]
                 |
          ┌──────┼──────┐
          |      |      |
       [Сын]  [Дочь]  [Дочь]
```

## Галерея фотографий (в разработке)

### Описание
Возможность загружать и просматривать фотографии человека.

### Структура данных
```javascript
photos: [
  {
    id: "photo_1",
    url: "base64 или URL",
    caption: "На даче, лето 2010",
    uploadedAt: 1640000000000
  }
]
```

### Планируемый функционал
1. Загрузка фотографий через file input (конвертация в base64)
2. Галерея с превью
3. Модальное окно просмотра в полном размере
4. Подписи к фотографиям
5. Порядок сортировки

## Галерея видео (в разработке)

### Описание
Возможность добавлять видео (ссылки на YouTube, Vimeo и др.).

### Структура данных
```javascript
videos: [
  {
    id: "video_1",
    url: "https://youtube.com/watch?v=...",
    caption: "Юбилей, 2015",
    uploadedAt: 1640000000000
  }
]
```

### Планируемый функционал
1. Добавление ссылок на видео
2. Встраивание YouTube/Vimeo плееров
3. Подписи к видео

## Комментарии (в разработке)

### Описание
Возможность оставлять комментарии и воспоминания на страницах памяти.

### Структура данных
```javascript
comments: [
  {
    id: "comment_1",
    author: "Иван Петров",
    role: "Ученик",  // опционально
    text: "Самый лучший учитель...",
    date: 1640000000000,
    likes: 5
  }
]
```

### Планируемый функционал
1. Добавление комментариев (имя, роль, текст)
2. Список комментариев с датами
3. Лайки к комментариям
4. Модерация (удаление)

## QR-код для надгробий (в разработке)

### Описание
Генерация QR-кода со ссылкой на страницу памяти для размещения на надгробиях.

### Планируемый функционал
1. Кнопка "Сгенерировать QR-код"
2. Скачивание QR-кода в различных размерах
3. Инструкции по изготовлению таблички

## Экспорт в PDF (в разработке)

### Описание
Экспорт страницы памяти в PDF-документ для печати.

### Планируемый функционал
1. Кнопка "Скачать PDF"
2. Красиво оформленный документ с фото, биографией и timeline
3. Опции: с фотографиями / без, формат A4/Letter

---

## Приватность (в разработке)

### Описание
Настройки приватности для страниц памяти.

### Планируемые уровни
- **Публичная** — видна всем
- **По ссылке** — только по прямой ссылке
- **Приватная** — только для владельца

### Планируемый функционал
1. Выбор уровня приватности при создании/редактировании
2. Пароль для приватных страниц (опционально)
